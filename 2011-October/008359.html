This is both a design and technical choice.

There's no existing technical structure for easy NPC persistence (persisting user identities is not the same).  So it 
would take a ton of work to create and I would have to predict what people would need and how they would use it.

Recreating the avatars as required in scripts, as Marcus discussed, is far easier.  It allows people to do exactly what 
they need and if a very common solution emerges then server support could eventually be added.
