One issue with this is that core is already a complex and unusual software product with >300K lines of code.  It's 
already difficult to understand and maintain - adding more code for divergent use cases would exacerbate these problems.

Hence the desire to eventually see downstream distributions (such as Diva's) that integrate OpenSim and other packages 
(such as profiles and web front end) to meet different use cases.

Whether this is realistic or not remains to be seen.  It might depend on whether parts of OpenSim such as agent and 
object management can eventually be considered generic facilities, analogous to webservers and database systems, or 
whether OpenSim is more like a complete content management system such as Joomla or Drupal that comes with a functional 
set of core modules and a wider universe of external modules.
