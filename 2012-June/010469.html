On Wed, Jun 6, 2012 at 3:36 PM, Nebadon Izumi <nebadon2025 at gmail.com> wrote:

> Default Max size allowed in OpenSimulator is 256m limits lower than that
> are generally because of the viewer you use, the stock LL and some of the
> TPV viewers only allow for 64m prims max, you have two options, you can
> find another viewer such as Imprudence, or Singularity that allow you to
> create larger than 64m prims, or you can use the simulator console command
> "edit scale", now before you run off and use this command there are a few
> things you need to be aware of, this function uses the prim name to scale
> it, so it is very important the prim you are resizing has a unique name,
> never scale "primitive" it will be a disaster, make it very unique like
> "resized_prim_0001" or something, ie > edit scale resized_prim_0001 256 256
> 256 < or ie : edit scale <prim name> X Y Z
>
> hope this helps.
