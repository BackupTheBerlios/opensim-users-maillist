The more I look at it, the more I think it needs a redesign more
than just a translation. One thing is, it uses ini-files in the webpath
to store sensitive information, the second is it seems to require access
to the config-files of the simulator. While not a problem if the
software and the manager-gui are on the same server, but this is not
always the case, esp. in larger grids. However, I believe that many
things done in the GUI can be used as an example. 

For a first step, I
would make the gui access remote sims either trough the REST-Gui or
using the Remote-Console. In a second step a client component running on
the remote sim could be added to handle direct access to the
configurations and other files. 

Regards, 

Klaus
