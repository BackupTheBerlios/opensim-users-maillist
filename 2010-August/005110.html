Hi Kenneth,

If you do not wish to use SQLite for region storage (I heartily agree with
this decision), you will need MySQL installed on your server. Additionally,
you will need .NET 3.5.

However, I would point out that this has no bearing on asset storage; as you
surmise in your previous message, assets will be stored on OSGrid's asset
server (or whatever grid you attach the region server to).

This should not be confused with region-side SQL storage, which does not
store assets, inventory or user information; rather, it stores only
region-side data, such as rezzed objects, their contents, and terrain
details.

Hope this clears up any vagueness in that respect :)

Cheers and Best Regards,
