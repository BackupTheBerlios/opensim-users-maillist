Actually, afaik whether 'hacked god mode' is available depends on the loginuri containing 'agni' or not. What it does, is that it side-steps local permissions checking, and allows all local changes, as well as sends all updates to the server, regardless of whether the client have perms or not. It is then up to the server to do proper perms checking and stop the request.
 
We actually discovered quite a few instances of the default server perms module not stopping some requests over hacked god mode a couple of months ago. So it's probably a good thing to test properly and in depth.
