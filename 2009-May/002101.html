On Thu, May 7, 2009 at 8:58 AM, Justin Clark-Casey
<jjustincc at googlemail.com> wrote:
> For those following SVN trunk (which I'll remind you is a dangerous 
> and risky practice), revisions r9438 (for sqlite) and r9440 (for 
> mysql) migrate the region data to use the dashed uuid format common
elsewhere (and already used for some attributes in the region data).  This
is necessary for future work.
>
> THIS MEANS THAT YOU *MUST* RUN prebuild.bat/prebuild.sh *AND* do a 
> clean build ("nant clean build" if you're building on the command line) IN
ORDER TO RUN THE MIGRATION.
>
> If you don't do this, then after updating:
>
> * On SQLite and MySQL you will see the completely blank startup 
> islands instead of your usual regions.   DO NOT PANIC ALL YOUR DATA IS 
> STILL THERE!  Simply do a runprebuild and a clean build as above.  This
will allow the data migration to run and make your region accessible again.
>
> * MSSQL is unaffected.
>
> Please spread the word about this as necessary.
