On 10/12/12 22:51, Rick Anderson wrote:
> I get this occasionally where a region is black listing a region in my own server farm. The error is lie the following:
>
> 22:42:43 - [WORLD MAP]: Blacklisted http://174.129.199.66:9000/MAP/MapItems/12093528396723200
>
> Any reason regions would be black listing each other? Is there a way to reset or un black list them?

I'm not an expert in this area, but afaik a simulator will blacklist another when it can't contact that other simulator 
to get map item data in response to a map request.  It would be odd if it's happening on a local server farm.

The timeout will be removed after 600 seconds.  This can be adjusted by changing BlacklistTimeout in [Startup].  There 
are currently no console commands to display or manipulate the blacklist, though I think these would be nice to have.
