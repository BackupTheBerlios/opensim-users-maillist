here are set of my assumptions. Let me know if they agree with
your understanding:

1. The region server handles a) all asset management for assets that 
exist in,
    created-in or modified-in that region, b) all agent management of who
    is located in the region, and c) all messaging that occurs in that 
region

2. The region server communicates to the grid server(s) when there is an 
agent
    or asset region crossing (in or out of the region) or when there is 
a teleport.
    Messages that occur near region boundaries communicate back to the
    grid server so that the grid may inform neighboring regions.

3. Inventory management of which assets are owned by agents is handled on
    the grid server side. The region server communicates back to the 
grid servers
    when there is a change in inventory (new item or removed item) while the
    agent is in that region.

4. User management is handled by the grid server.
