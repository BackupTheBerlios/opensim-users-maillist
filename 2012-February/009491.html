Yes, there is a known problem with OpenSimulator 0.7.2 shutting down on Mono 2.6.x when the number of scripts pass a 
certain threshold.  It's an issue with unloading them from the AppDomain.

I don't believe you actually need to worry about this problem - by the time OpenSimulator reaches this point in shutdown 
everything that needs to be saved has already been saved correctly.

Upgrading to Mono 2.10.x and/or the forthcoming OpenSimulator 0.7.3 will make this crash go away.  In the former case 
this is because Mono fixed the bug with unloading AppDomains and in the latter because OpenSimulator no longer bothers 
to unnecessarily unload scripts if the simulator is completely shutting down anyway.
