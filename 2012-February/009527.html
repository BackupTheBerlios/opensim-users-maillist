Hi everyone

Last week we encountered two errors with our standalone simulator:

i) Certain existing users were being prevented from logging in and
were receiving the message "Unable to create login request" from
within the client. The problem turned out to be down to a bug in
MySQL. This was solved by applying a fix within StandaloneCommon.ini
which was to append the string Old Guids=true; to the end of BOTH
ConnectionString lines.

ii) With a lot of objects/scripts rezzed we encountered segmentation
errors when shutting down the simulator. The solution was to upgrade
the version of mono we were using to 2.10.2

Yesterday afternoon  the simulator crashed (which it hasn't done for
ages) and the error message in OpenSim.log is appended

Could this be anything to do with the upgrade of mono or the changes
to StandaloneCommon.ini?

Any ideas welcomed please

Thanks very much
