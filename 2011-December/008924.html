If you're using the latest osgrid distro files, you may want to try commenting out the lines

     Cap_WebFetchInventoryDescendents = "localhost"
     Cap_FetchInventoryDescendents2 = "localhost"

in [ClientStack.LindenCaps].  Having these HTTP inventory capabilities enabled usually results in much better 
performance with large inventories, but there are some bugs yet to be fixed which cause problems under some 
circumstances (e.g. attachments not appearing and being registered at 'invalid positions').  I meant to take a look at 
these issues last Friday but unfortunately the complete holiday brain shutdown kicked in early :)

Disabling these capabilities will cause the viewer to fallback to the older UDP inventory mechanisms which don't have 
these issues.

If you're not running these settings then I don't know what the issue might be.
