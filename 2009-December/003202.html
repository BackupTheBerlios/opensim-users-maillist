Isn't is clear that that was a "locales" issue ?
We already had that kind of problem on linux/mono a long time ago ( more than one year ).

This "locales" problem affect the whole OpenSimulator when not set to en_US.UTF8 ...

On my side, changing the locales to en_US.UTF8 on my Linux box solve many issues at a time :

- logout position stored into the MySQL agent table,
- avatar height stored into the MySQL appareance table.
