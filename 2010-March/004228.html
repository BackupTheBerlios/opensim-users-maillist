One of the problems with the BVH format BTW is that there is no set of
standard names for the joints.

Second Life and presumably OpenSim use "pretty much" the Poser set while
dropping the hands, feet, and head, jaw and eyeball data. This is workable
for a one way transfer, but to restore a Poser/whatever BVH file these have
to at least be considered.

issues resolved.
