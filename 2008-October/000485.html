The master avatar is somewhat of a remnant from the very early days, where we had no permissions or access control (some would argue we still haven't) so, it's basically the 'god' of a region, overriding most (all?) permissions. Of course, you can set the same master on several regions, making it the 'god' of a cluster of regions.
 
Standalone has an internal user service, so if you were using standalone, you had to specify avatar name and password in the ini, in order for the master to be set up if there was none.
I'm actually unsure as of the state and future of the master avatar; I believe it to be an historical oddity, on its way to obsoletion as we get groups and real acl, but others might have differing views.Best regards,Stefan AnderssonTribal Media AB Join the 3d web revolution : http://tribalnet.se/
