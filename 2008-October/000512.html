Stefan Andersson wrote:
> Codifying a UUID is considered a "bad thing"(tm). The right way to go is
> to never trust (rely on) a given guid, only to trust (rely
> on) internally assigned guids.
>  
> So, if grid A tells grid B that it has guid X, B could store this data,
> but will store the data with the internally generated guid Y as key. And
> whenever it refers to it internally it will refer to Y, not X.

+100 on that.
