On Fri, Apr 3, 2009 at 9:09 PM, Charles Krinke <cfk at pacbell.net> wrote:

Coitainly.

As I think about it, there are a few parts. One is OpenSim packet
processing to enable the groups messages. Another is a connection to a
MySQL (or MSSQL) database to hold the messages.

In thinking about "groups", it seems to me that we can start small.
After all, we really only need to make sure the packet processing is
worked out and we can store a few messages in a MySQL table.

Personally, I would be delighted to see, say 1) 10 groups, 2) Max number
of messages of say 100 per group. 3) Max length of each message, say 256
characters.

That would get us off the ground and flesh out a complement to our
current IM implementation in OpenSim, methinks.
