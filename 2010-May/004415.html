Note that for setting up grids where all simulators belong to the same 
authority, ppl don't need to use any servers whatsoever other than a SQL 
server and the simulators themselves. The "grid mode" with its resource 
serv*ers* has been popularized because of OSGrid; but OSGrid is a very 
special case -- each simulator belongs to a different person; it would 
be very unwise to share the MySql DB password with everyone.

When all simulators are under the same authority, you may as well let 
them access the DB directly. It simplifies things by a lot. MySql 
servers are mature pieces of software that can take a lot of heat 
without breaking a sweat.

Information about serverless grids:
http://www.metaverseink.com/blog/?p=22
(I recommend turning mega-regions off for this)
