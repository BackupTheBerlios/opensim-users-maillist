A follow up on this problem.

I have just spent some time with Tom Meta of the M7 support team and we seem
to have found the root of the problem.

I used Second Inventory to import these items from my private standalone sim
where I do all of my builds.

It appears that Second Inventory generates bad perm fields when it imports
objects and this causes problems with the perms in world when the object is
rezzed, transferred etc.

The work around is to rez the object and then take the object using the M7
(or equivalent) viewer - this fixes the problem about 80% of the time. For
wearable objects (e.g. hair) this requires that you redo the attachment
point. :-(

If I try to backup the imported objects from Meta 7 into SI, it exports
fine. However I am unable to re-import the item to Meta 7 or elsewhere due
to copyright (permissions?) errors - even to the same M7 account that I used
for the backup!.

This was not a huge problem before the SL TPV policy, but perms checking is
now much more aggressive than it once was even for stuff that never uses
Second Life!

I have put in a bug report to Second Inventory, please see their website for
more information.

Karen
