UInuYasha (and others),

I've created a new viewer, based upon the Emerald sources. Although this
viewer was created primarily to support the display of Youtube movies inside
Second Life, I also wanted to use it for OpenSim and ran into the same
problems as you.

The viewer I created, is changed slightly, so that regular textures do
download and display succesfully, However, there still is an issue with the
baking of and uploading of textures, which causes issues in combination with
Opensim. I believe these issues are caused by the use of the Emerald
specific 'kakadu' libraries (emkdu.dll). To 'work around' the baking issue,
rename the emkdu.dll file, so that the viewer can't load it during startup.
It will then use either OpenJpeg libraries or llkdu.dll (if available).

The viewer can be found on the TEKSTUFF website: http://www.tekstuff.net/

Make sure you have Emerald installed already, then download and install the
TEKSTUFF installer, and start the viewer using the TEKSTUFF shortcut. It
should use all the current settings from the Emerald installation.
