Yes I can definitely see where having a reduced "needs only" ini file would
be nice, but I am a command line kind of guy as I am sure many of the NIX
people are thus copying and pasting between files is a pain for us sometimes
so it is pretty nice to have it all in one OpenSim.ini file. Generally in a
NIX environment a server does not run any type of Xwindows layer to keep the
overhead down since we usually shell in, do our thing with vi or some text
editor and exit off. How NIX admins approach their systems can be very
different than Windows admins. What about a web based interface as a
compromise? Most of us run web servers for the login, support, helper etc.
pages anyway. Just a thought.

-Robert
