Tom,
  That looks like the debug messages from TestClient. You might also post this question to the libomv list. 

It looks like someone else just posted a similar question to the libomv group yesterday:
http://groups.google.com/group/libomv-dev/

I use TestClient extensively and see that message all the time in my logs but do not get 'cloudy' avatars. I think it's a red herring. I believe there is a texture in the default avatar which is corrupt but which the Linden-derived viewers handle gracefully/differently. My suggestion is to create a new user, log it in with a graphical viewer so that a stable appearance is saved on the server, and then use TestClient to reconnect that same account. Does that avatar still show as a cloud?

~Dan
