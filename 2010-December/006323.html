You mention a router, is the router also the gateway to the internet? If the
gateway is a separate device, then it is possible that the issue is between
the router and the gateway. I have both a router and gateway and needed to
make sure both had the proper ports set, or that the router was a DMZ for
the gateway before I could get both external and internal users to access my
test grid.

The main thing is to make sure that the local user appears to the system as
an external user. So if you changed the routing for the local user to
prevent them from actually bouncing their traffic from the internet back
into the system it may be preventing the return trip and so your
authenticating might be getting screwed up. I'd look to make sure you don't
have the internal IP of the server used on the local machine, nor any IP
tables on the router that shortcut the network loop by sending the traffic
to the server instead of the external IP.
