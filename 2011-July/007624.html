  I now have success and found a very obscure article that I think should be made more prominent.

  It states that all mega-regions need, as their starting or home set location, to be the south-west corner sim. This solved my problems and all is working well now.

  On Sat, Jul 2, 2011 at 10:04 PM, Henry Wills <fubat.enterprises at gmail.com> wrote:

    Not sure if this helps, but what I did was create a new region, then
    delete the old regions from the console. I shut down, deleted the
    RegionConfig.ini file and renamed the new one to RegionConfig. I also
    left the ContiguousRegions setting to true. I then added new regions,
    which automatically created .ini files for each new region. Once the
    region .ini files were created, I copied and pasted the info into the
    RegionConfig.ini file I had created. I had no problems with up to 16
    regions joined as one megaregion. I did start running into some lag,
    but that is due to my system being NQPE (Not Quite Powerful Enough).
    Its a single-core processor, though with 3Gb RAM and a nVidia GE9500
    card.
