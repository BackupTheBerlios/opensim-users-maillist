Glad to see you finally got it all going :)

It is unfortunate that Mumble can only be directly configured to one Region but it is great that once the murmur server has authenticated you, you can move between regions on the same grid with voice capability following you. The workaround at the moment for this obviously is to ensure users log into the voice-enabled region first. For existing users, you will need them to set their home position to that region and to enter the grid via their home location. Setting the following may help also to ensure new accounts use that region as the default home region (well, It can't hurt..):

[GridService]
;
;
;
;; Next, we can specify properties of regions, including default and fallback regions
    ;; The syntax is: Region_<RegioName> = "<flags>"
    ;; where <flags> can be DefaultRegion, FallbackRegion, NoDirectLogin, Persistent, LockedOut
    ;; For example:
    Region_Welcome_Area = "DefaultRegion, FallbackRegion"

"Region_Welcome_Area" needs to be replaced with the Region for which you've enabled Mumble. 

This is located depending on your setup:

Standalone: %opensimdir%/bin/config-include/standalonecommon.ini
Grid: %opensimdir%/bin/Robust.ini

With a bit of creativity, you can make your "Welcome Region' not only one that activates voice, but also act as an exciting gateway to your grid :)
