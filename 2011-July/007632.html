Well this has been difficult to get going for me as well so don't feel too
bad.

I do have it working for imprudence 1.3.0 RC. (I didn't get  it to work for
a later version) and diva build opensim 0.1.1.1 Post_fixes  

1.      On each user's machine running imprudence the only thing I did is
install whisper 0 .2.7 version found here
http://whisper.vcomm.ch/download/0.2.7/whisper_setup_0.2.7.exe I did not
modify any files, just installed this package in the imprudence folder.

2.      On the machine running opensim I put the 3 binnaries we both got
from the other fellow into the opensim/bin folder and then added the new
section to my opensim.ini file. I did not make any changes to any other
files. Here is a copy of what I added to the opensim.ini (remember that the
name of the region is case sensitive and no special characters.) If you
visit http://3dcolab.com:9015/wifi/ you could get an account and see if your
imprudence set up is working on a our world which I have it working on.

3.      I am not sure it is working quite correctly and I also point out
that you need to pay close attention to how your pc input and output is
configured. Also it does not work well for me with out a head set. Let me
know how it is going.

 

 

[MurmurVoice]

   ; New voice daemon which will do speaker indication,

   ; positional voice, and high quality audio codec. Host voice on the

   ; same[MurmurVoice]

   ; New voice daemon which will do speaker indication,

   ; positional voice, and high quality audio codec. Host voice on the

   ; same server, or use Blacier2 to proxy the insecure ICE protocol.

   enabled        = true

   ; Hostname to send the user

   ;murmur_host    = <your server's ip that outside client need to reach>

   murmur_host    = 67.227.131.4   

   ; Server Identifier (normally 1)

   murmur_sid    = 1

   ; Endpoint to system running mumble

   murmur_ice    = tcp -h 127.0.0.1 -p 6502

   ; Endpoint to this system (not needed for glacier2)

   murmur_ice_cb    =tcp -h 127.0.0.1 -p 6503

   ; Parent channel name to use

   ;channel_name     = <the name of the region that you are enabling voice
on>

   channel_name     = 3dcolabDiva1

   ; Enable use of glacier2 (ignore murmur_iec_cd if false)

   glacier     = false

   ; Glacier settings will not be taken into account

   ; Glacier proxy

   ;glacier_ice     =Glacier2/router:tcp -p 4063 -h <your server's ip that
outside client need to reach>

   glacier_ice     =Glacier2/router:tcp -p 4063 -h 67.227.131.4

   ; Glacer session user

   glacier_user    = someusername

   ; Glacier session passsword

   glacier_pasds    = password     server, or use Blacier2 to proxy the
insecure ICE protocol.

   enabled        = true

   ; Hostname to send the user

   'murmur_host    = <your server's ip that outside client need to reach>

   murmur_host    = 67.227.131.4

   ; Server Identifier (normally 1)

   murmur_sid    = 1

   ; Endpoint to system running mumble

   murmur_ice    = tcp -h 127.0.0.1 -p 6502

   ; Endpoint to this system (not needed for glacier2)

   murmur_ice_cb    =tcp -h 127.0.0.1 -p 6503

   ; Parent channel name to use

   ;channel_name     = <the name of the region that you are enabling voice
on>

   channel_name     = 3dcolabDiva1

   ; Enable use of glacier2 (ignore murmur_iec_cd if false)

   glacier     = false

   ; Glacier settings will not be taken into account

   ; Glacier proxy

   ;glacier_ice     =Glacier2/router:tcp -p 4063 -h <your server's ip that
outside client need to reach>

   glacier_ice     =Glacier2/router:tcp -p 4063 -h 67.227.131.4

   glacier_user    = someusername

   ; Glacier session passsword

   glacier_pasds    = password
