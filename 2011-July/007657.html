On 10/07/11 03:12, Chris wrote:
> Hi all. I just recently copied my OpenSim installation over to my Linux machine (for testing purposes) and am running
> into a curious error after I've loaded my IAR and OAR backups to the database.
>
> I will get a large rash of these errors as my inventory loads:
>
> [LOCAL INVENTORY SERVICES CONNECTOR]: Could not find item with id <random UUID here>
> [LOCAL INVENTORY SERVICES CONNECTOR]: Could not find item with id <random UUID here>
> [LOCAL INVENTORY SERVICES CONNECTOR]: Could not find item with id <random UUID here>
> [LOCAL INVENTORY SERVICES CONNECTOR]: Could not find item with id <random UUID here>
> [LOCAL INVENTORY SERVICES CONNECTOR]: Could not find item with id <random UUID here>
>
> etc.
>
> Searched for the items in my inventory using Phoenix's search by UUID filter, and while it may come up with some partial
> matches in my inventory, it doesn't come up with an exact match once the entire UUID is typed in the search box.
>
> I know that when a new user is created it will show this error 4 or 5 times due to OpenSim not creating default clothes
> or body parts but I've counted about 20 + errors as my inventory loads up. These errors do not seem to show up (more
> than just the usual 4 or 5) when running under Windows.
>
> When comparing the number of items in inventory on Windows OpenSim installation to Linux OpenSim installation they
> appear to be the same number of items.
>
> Is this something I should be concerned with?

I'm not totally sure why this would happen.  If I had to guess, then I would suspect broken inventory links if the IAR 
has been taken from a viewer 2 inventory.  The target of an inventory link probably isn't being restored properly on IAR 
load, leading to requests for item ids that don't exist when the links are dereferenced.

If this is the reason then it's fixable and I've made a note of the potential bug.  Thanks.
