With all likelyhood the answer is never. 

The trunk isn't meant to contain all the different modules for all possible uses of opensim. We actually want to go in the other direction and in time move some of the modules that are currently in the trunk out onto the forge. 

The general idea of the opensim project is for it to be just a core platform that various uses/solutions can be built around. But not to provide all those solutions in one self contained project. There are so many different uses that we hope people will in time use opensim for that, we just can't provide one generic out of the box solution for them. 

What we most likely will see in time, is a opensim installer that can display a list of what modules are available for opensim and then if the user wants to use a module, the installer could download it and install it automatically.
