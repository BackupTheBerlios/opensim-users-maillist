Ralf asked, "is there a way to make " link-region <Xloc> <Yloc>
target" permanent or put it in opensim.ini?"

I used to use a "startup commands" script with a bunch of
"link-regions," but was unhappy with all the connection activity at
startup.

What I have done now is build a hypergrid transporter. It lets you
scroll through map images of valid hypergrid destinations. When you
see the one you like, touch the platform, and it dynamically issues
the console link-region command and then sends you off. Feel free to
try it out at sim.thestudyofracialism.org 9000. Its interface is that
of the short-range teleporters in Stargate.

Oddly, Diva's latest change also changed its behavior. The device uses
a sensor pulse to teleport everyone who happens to be standing on the
platform when it is triggered. Before Diva's latest change, it shipped
everyone off at the same time, and they would all arrive at more or
less the same time. But now it sends each person one by one, so the
last guy on the platform has a bit of a wait.

On another subject entirely, Melanie recently added the ability to
read the ini file from a URI. "If the -inifile option is a http://
URI, it will be fetched and parsed as an XML config." Does anyone have
a sample XML showing what this should look like?
