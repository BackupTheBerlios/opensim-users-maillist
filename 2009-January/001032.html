The current discussion about writing notecards is no doubt connected to the 
common practice of storing configuration data in notecards. realXtend 
provides an interesting free-form data panel for every object which can be 
used extremely effectively to store configuration and state data. As the 
rexmodule for Opensim stabilizes, Opensim developers and users may prefer to 
use this mechanism for configuration data in situations where no script 
compatibility with Second life is required.

I've posted a tutorial on using the data panel in conjunction with the 
Python CompObj library at 
http://peterquirk.wordpress.com/2008/12/31/tutorial-realxtends-powerful-configuration-management-for-models-without-notecards/
