In any case:
	1° Define if you want to use your server from outside your network
		- if yes, the valid address is your external IP. You're likely to use a dynamic or fixed dns service to reference it
		- if no, the valid address is your internal IP (192.68.* or 10.*)
	2° In Robust.HG.ini or Robust.ini, replace ALL occurrences of 127.0.0.1 by your server address (as defined above)
	3° In GridCommon.ini, replace ALL occurrences of mygridserver.com by your server address

(in most case, you'd keep "localhost" as is)
	
If you run for outside world too, 
	4° Set your router NAT accordingly (at least ports 9000 and 8002 should redirect to your internal IP address, tcp and udp)
	5° In Regions.ini, replace SYSTEMPIP by your server address

IN NO CASE should any 127.0.0..1 or gridserver.com remain in your .ini files!

Next indications should be followed only if you still get problems:

Most router would behave correctly with internal access to external IP.
Some don't.. In some cases, you would need to cheat your local machines

Only if needed, add following line in /etc/hosts file or it's Windows equivalent
	192.x.x.x	your.dynamic.hostname
(replace 192.x.x.x with your internal IP address)		
(on your robust, regions and client machines)


That said, as Fleep told, if you don't need to run more than one region server, you better choose a standalone setup. But the same considerations apply.
