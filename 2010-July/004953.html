In these situations it's also worth trying a relog.  Objects can 'go missing' when the initial object update information 
between client and server gets lost.  This can sometimes be alleviated by lowering the network bandwidth that the client 
tries to use (this is located in the preferences).

In fact, I'll be surprised if there are actual persistence problems.  However, objects are persisted to the database 
only after a set period of time (60 seconds, afaik).  If you lose power before this happens then the objects will be lost.
