Hi,

Just want to inform about a new bsd-licensed socketed remote service 
tool(currently only for linux) called Teknon(http://hwios.org/wiki/Teknon). 
The idea is that on every host you run opensim services on, you also install a 
teknon client and define services in it. All the clients connect to a central 
Teknon server, that keeps track of clients and services. From the server 
console you can start/stop/kill or send commands to all services that are 
defined by the clients. All client subprocess console feedback is sent over a 
socket to the server, so you can keep track of what's going on in the 
consoles. 

For now it only has a console connector for the server, but a webbased 
connector using websockets is next(for HWIOS integration). It's written on top 
of Python/Twisted PB and it's kinda functional already. Please let me know 
what you think about it. The app can use some testers, feedback and maybe 
people who want to help developing it. Have fun!
