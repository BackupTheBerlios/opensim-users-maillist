Opensim is modular, has an add-on system. It does not know anything 
about the modules that you will use. It simply loads the modules you 
have installed, from the separate files that it finds in the add-ons 
dir, at startup.

> the problem is "Reference Path"  to physics is not 0.7 compatible i think  I
> will try with 0.6.9
>   

I think Mikko P. has the information somewhere about which branch of 
modrex works against which version of opensim. IIRC the idea is that 
master modrex tries to track master opensim, but it sometimes takes time 
to catch up. So there are tags of versions that work against some 
certain version of Opensim.

If you just want to use it, and not develop new features to modrex or 
something, I guess easiest is to use a release with everything prebuilt 
and  configured. Modrex is included in the Taiga distribution of 
Opensim, http://wiki.realxtend.org/index.php/Getting_Started_with_Taiga 
-- that uses 0.6.9 still 'cause not everything works with 0.7 yet (but 
Mikko said recently that most things do work already with some 0.7)
