> 2/ Spam is high. How can we help?

I know there are all kinds of tools for trying to control spam (captchas, etc.) but these haven't worked well in the 
past.  I don't think there's any way we can go back to automatic user account creation - it just allows too much spam.

However, I don't think that asking people to request accounts on this mailing list is a good solution either.

At the moment, I'm thinking that perhaps having a moderation queue for new user accounts fronted by some basic 
OpenSimulator questions (e.g. Is OpenSimulator a) a car, b) a dog, c) a server).  That way, even if someone does take 
the time to program a bot to answer the question correctly, at least it's just the moderation queue that will get filled 
rather than the wiki.  However, I do want to be very cautious in not letting a ton of spam in again - it might fry Neb's 
brain!

Of course, this relies on people looking at the moderation queue regularly, which is the costly thing.  If anybody would 
be happy to help with this duty that would be great.

Or if you (or anyone else) has practical spam fighting experience and can suggest a better alternative actiomn then that 
would be great.
