On 17/04/13 05:22, Tom wrote:
> I am trying to gain external access to my OS standalone server "Diva".  On the internal network all is fine even using
> my external url (via dynamic dns). The  External login gets as far as [SCENE]: Region Central authenticated and
> authorized incoming root agent XXXXX (circuit code 230241706) (using the rest console).  Firestorm then fails giving the
> message "Waiting for region handshake" after a number of attempts.
>
> The IP address is redirected using dynamic dns, the host machine has a reserved IP address.  Port forwarding for both
> TCP and UDP are set to 9000..9004 to this machine is set up. I have the same problem with an without the firewall on
> (but forwarding set).
>
> I am able to access the diva Wifi, webpage and use the rest console for this machine. ssh works fine.
>
> Any ideas?

This usually happens if the ExternalHostName/InternalPort combination from the Regions.ini file that the login service 
gives to the viewer cannot be reached by the viewer.  The naming is confusing here.  You need to check that external 
viewers can reach the region addresses:ports being offered.
