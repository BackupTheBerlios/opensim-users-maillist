I don't know how many of you try out the bleeding edge (git checkouts), but
something bad has happened with the opensim physics as a result of 
apparent changes
to the physics API sometime in between 10/19 (the checkout where the 
flailing
stops, and the ODE physics is quite stable) and today 10/28.

A checkout as of an hour ago with the default ODE settings (except for
av_capsule_tilted = false) has the avatar:

 * not being able to walk through double, open, doors
 * bouncing when page-down  (or C) is pressed so that the avatar meets 
the ground (the avatar
    used to correctly "stop flying" and land on the ground). Instead, 
the avatar never stops
    flying and just bounces back up.
 * walking through (instead of up) stairs
 * walking horizontally on air (for some time) instead of down stairs

I may eventually file a mantis, but wanted to see if others experienced 
this.

System: Linux 64-bit
