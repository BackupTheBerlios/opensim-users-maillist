2.4.2 is in Ubuntu Karmic.

I choose to follow this branch and while mono itself seems quite stable, they 
have been fiddling with the boot system recently and I have sometimes been 
left with a system that won't even get to a command prompt without some 
hacking via my reserve boot partition  (not complaining here, just warning 
people - that's what alpha releases are for, and I accept that ;-) ). If you 
can get away with just pulling down the needed mono packages you should be 
fine - just watch what dependencies it tries to pull in with them!

Karmic should become stable some time next month if Cannonical's release 
timetable holds.

> >
> > This has become necessary because the bug is rather difficult to
> > isolate and therefore impossible to work around reliably.
> >
> > We do understand that this may cause difficulties for some people,
> > who have been relying on packaged binary versions of Mono. Version
> > 2.4.2 is recent enough to have no binary packages, so it will be
> > necessary to build Mono from source. However, we feel it is
> > unavoidable at this time, as we can't recommend or endorse a Mono
> > version that we know to contain a showstopper bug.
> >
> > On the upside, 2.4.2 has much better performance and stability,
> > compared to 2.0.1.
