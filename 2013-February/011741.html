The word "corrupt" is perhaps rather unfortunate here.

The "appearance show" command checks to see if each uploaded baked texture is present for the avatar and if it is, it 
runs it through JPEG2000 decoding.

If any required baked texture is either missing or fails JPEG2000 decoding, then the appearance is reported as 'corrupt'.

When the viewer teleports to another sim, the baked textures are missing (because opensimulator does not place these 
temporary textures in the asset store by default) and usually this triggers the viewer to reupload them.  However, for 
some reason this may not be happening in this case.
