Not sure, but let me give some detail on what I did.

I built a starship on my offline sim quite some time ago, maybe a year ago
and backed it up as an XML using Vista and SoaS and before scripting, I
think the OpenSim version was something like 0.7.1. I then switched over to
checking out AuroraSim 0.4.3. (I am running both AS and OS at home) No
problems importing it then, textures came through fine. Recently, saved the
AS sim as an OAR and it was uploaded on Created Worlds, which is now
running AuroraSim 0.5.1.

I know, this is an OpenSim users mailing list, I'm just giving the
background.

Anyway, that's when I found that half the textures had been blanked. Thing
is,only some of the same textures got blanked and that texture had been
imported as an IAR. My inventory copy was fine. I double checked my offline
AS sim and found blanked textures. I thought it might be a problem with the
latest AS release. So, I went to my offline OpenSim setup and imported the
original backup of the linkset from a year ago. The same prims were
blanked. It seems to me that the common point of these is that I had
exported the original using Vista-32bit and am now running Win7-64bit.

Its an annoyance to have to re-texture, but I can deal with it. After all,
it* is* beta'ware. [?]
