Ah that is an interesting point - I checked and sure enough it is only the
root prim textures that are vanishing.  So a work-around is to restructure
the HUD to have a small transparent root prim.  But I'd like to try to see
what is going on - I noticed reading an old description of the viewer
architecture that textures are assigned a boost value - HUDs should be
assigned fairly high and selected (clicked on) textures get very high too.
 One possibility is that the root prim in the HUD isn't always getting this
boost value set correctly so other textures in the scene are taking
precedence.  Vague idea I know...

Thanks for the root prim hint!

Neil
