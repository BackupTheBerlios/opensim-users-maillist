Hi Kay,

We only got the system up and running a few months ago so I'm not sure I
can answer that yet, certainly nothing we've done so far has taxed the
server at all.  There are 30 regions on the system, but we don't have much
content built yet and the maximum concurrency has only been 15 users I
think.  It's on my to-do list to do some performance testing with content
load and maybe scheduling a concurrency test once we have some other work
done.

>From my experience with FleepGrid, I think there isn't really a simple
answer though.  It's not how many regions a server can support as much as
it is how many regions + how much content (builds + textures + scripts) +
how much concurrency + how much db load + some x factor I probably don't
even know about, and I think a lot of that is dependent upon how much RAM
you have available as much as processor power.  On FleepGrid, I was
surprised to see very little load as I put up region after region.  I
thought wow, this old box can support 12 regions, crazy!  Then I started
building..  and the load shot way up.

We haven't gotten enough content on the UCSIM grid to even make a real blip
yet, but I'd love to hear what other grid operators have found to be a sort
of average.
