I don't run Diva Distro, so I don't know if that can be run in grid
(ROBUST) mode?

If not, then you'll need to go to http://opensimulator.org/wiki/Download and
download the latest version of opensim and follow the instructions at
http://opensimulator.org/wiki/Configuration#Running_OpenSimulator_in_Grid_mode.
 That page also has some nice diagrams explaining the difference between
standalone and grid mode.  I could be mistaken, but I think Diva Distro can
only be run in standalone mode.. ?

Though I should say, before doing ANYTHING, make sure you take a good
backup of your database, and I personally take OAR snapshots of all of my
regions before I upgrade or test any new configuration.

Regarding "instances", I found it very confusing at first when people
talked about running simulators in different instances, I didn't understand
what that meant.  Essentially, you copy all your opensim files into another
directory, edit your regions.ini file in the new directory to only include
the regions to run on that "instance" (and of course edit your old
regions.ini file to remove the regions you're going to run on the new
instance), and then start the opensim.exe process in both the old and the
new directory.  So for FleepGrid, as an example, my file structure looks
like this:

-opensim_0.7.4
  - addon-modules
  - bin
  - doc
  - share
  - ThirdParty
  - ThirdPartyLicenses
  - WifiPages
-opensim2_0.7.4
-opensim3_0.7.4
-opensim4_0.7.4
-opensim5_0.7.4

In each of those directories, in the /bin/regions.ini file, I have one
region listed for that "instance", and then to start up the grid, I first
start ROBUST, then start the opensim.exe process in each of those
directories.  Does that make sense?

Feel free to ask more questions on the list, and if it's any help, I keep a
change log for FleepGrid with all the details and example ini files etc. on
my wiki at:  http://fleep.wikispaces.com/FleepGrid.

Good luck!
