I have 2 scripts - which crash the simulator. This scenario is for windows
64bit

When using opensim.exe they crash the simulator. 
When using opensim.32bitlaunch.exe they do not crash.

I found out that this must have something to do with the .net version
running on that server. 

On my other server running .net 3.5 those very same 2 scripts (after loading
a duplicate .oar)
do not crash when using opensim.exe or opensim.32bitlaunch.exe.

As i use the osgrid distribution, my GridGod Nebadon was so nice taking some
minutes to talk 
about this with me. He said that he/they compiles the distribution with .net
3.5 and it is 
very possible that when using it on .net 4 server these things can happen. 

Mayby take this into account when filing a mantis or checking for errors.
May solution 
Is either running opensim.32bitlaunch.exe for those regions or downgrading
to .net 3.5

The thing to try here is seeing if there's a difference between using
OpenSim.exe on 64 bit, which will run as a 64-bit process, and
OpenSim.32BitLaunch.exe, which will run it as a 32-bit process.

As noted later on in this thread, there is at least one bug where an object
can mysteriously crash OpenSimulator running in 64-bit mode on Windows but
not in 32-bit mode.  There may be more.

This is partly why I haven't yet changed the docs to advise people to use
OpenSim.exe on 64-bit Windows by default.  The same problems don't occur
with 64-bit mono.
