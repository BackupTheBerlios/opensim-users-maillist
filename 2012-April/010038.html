I am running Windows 7 x64 and the only problem I found is I may need a 
memory upgrade.  It is running much better since 0.7.3, except one 
little thing.  The new x64 ability allows it to use to much of my RAM.  
I am one of those people running 2024sq of land.  So far no lag, and 
only a couple of the regions with stuff built.  It I leave it alone 
running only the one instance of opensim.exe it leaves me with 1011MB of 
my 8GB.  I use Phoenix Firestorm when on my grid and that takes up a ton 
too.  I run a small gadget on my desktop that watches my RAM.  With 
Windows 7 x64 running and only those two items I get a warning telling 
me I have only 500MB free.

I am a bit worried that one I start building on the other regions my 
system will crash out with an out of memory error.  It has done that 
already when I tried to open Firefox (The browser) while both opensim 
and Phoenix Firestorm at the same time.  I am going to start saving for 
one of those new AM3+ bulldozer 8 core, machines with a ASRock mobo that 
can hold 32GB of ram, and hope my computer grows to fit the needs of 
opensim.  This might take me a year or two.

I just wanted to help with my two cents on the new opensim 0.7.3. 
running on Windows 7 x64.  I haven't yet tried uploading any OAR files.  
But I have noticed being logged out for trying to fly from one end of my 
2048sqm to the other.  Sometimes I make it, other times I don't.

Good luck.
InuYasha

> after loading a large OAR file that turned out to be (most likely) an

>

> http://opensim-users.2152040.n2.nabble.com/Intermittent-crashing-System-OutofMemoryException-td7281298.html

>
> The solution was to separate out the regions into multiple opensim 
> instances.
>
> This problem occurred with the 0.7.2 release, but I understand the 
> 0.7.3 release allows you to run opensim as a 64 bit process which 
> takes better advantage of memory resources, so not sure which version 
> you're running, but if you haven't upgraded to 0.7.3 yet, might try 
> that and see if the issues persist.
