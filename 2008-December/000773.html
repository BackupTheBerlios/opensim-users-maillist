Dear Generic,

I'm a little confused - that text doesn't appear anywhere on that wiki page, and never has.  I guess you're probably 
referring to some other page.

> 
> I prefer to describe SQLite as: 
> SQLite is a software library that implements 
> a self-contained, serverless, zero-configuration, transactional SQL 
> database engine. SQLite is the most widely deployed SQL database engine 
> in the world. The source code for SQLite is in the public domain.

That's true, although it's best suited to some needs and not others.  In particular, I suspect that multi-user access 
with large data sets is not one of its targets.

As far as I recall, the creator of SQLite himself pretty much said this on one of Leo Laporte's FLOSS podcasts.
