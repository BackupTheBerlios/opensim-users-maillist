Curious.  I'm using MySQL in grid mode, and as far as I can tell
estate_settings.xml not been modified by anything I've done since
installation, yet its modification date/time is from around the time
last night when I was changing terrain texturing details on one of the
four regions. The height data associated with terrain textures,
however, looks to be the default.

I've migrated the regions before, including a full reinstall when my
HDD failed, and only copied the following:

- OpenSim.ini
- Regions/*.xml
- OAR backup (through save-oar)
- MySQL backup (of opensim schema)

That went fine, and the regions were as I'd last had them, at least as
far as I know.  The UGAIM servers are OSGrid's, the platform Ubuntu
Linux 8.10.
