Jefferson,

This isn't a big deal at all.  Keeping in mind that all viewers have the 
option of logging you into 1 of 2 pre-defined locations, "My Home" and 
"My Last Location", which are controlled via preferences and/or a menu, 
most of us have one of these option set as default and the 'default' 
default is "My Home".  Having said this, a virgin OpenSim account has no 
home position set.  The exception to this is an account created by an 
API or web interface which may be configured to assign a specific 
landing point (region/parcel + x,y,z coords) to all new accounts.  The 
WiFi web interface included in the Diva Distro has this capability 
tucked away in its .ini file settings.

What you're seeing is OpenSim successfully creating the account but 
lacking any home position directives to add to the database.  The viewer 
then places you (upon first login) at the default 'fallback' region 
defined in the [GridService] portion of whichever common.ini file you're 
using.  Once you set a home position for the new account, this will no 
longer happen.

Alternatively, entering the name of an existing region in the 'Start 
Location' of the viewer prior to logging in will circumvent this issue 
as well.  Hope this helps.

.core
