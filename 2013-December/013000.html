> If your asset cache is being stored on disk, then the other suspect for
> this issue is for some reason not having
>
> Cap_UploadBakedTexture = "localhost"
>
> in [ClientStack.LindenCaps]
>
> Normally, this is set by OpenSimDefaults.ini but customized configuration
> load setups can get tricky.  The last time this cropped up on this list in
> October ([1] is the last reference, unfortunately a bunch of e-mails went
> off-list), it turned out to be that a grid config was using -inifile and
> -inimaster switches in such a way as to bypass OpenSimDefaults.ini.
>
> You can see the actually used config with the various config console
> commands [2]
>
> In theory, not having the above setting should make viewers fallback to
> the UDP load mechanism.  However, I'm beginning to think there may have
> been a regression in OpenSimulator 0.7.6 which stops this working in some
> circumstances.
>
> [1] http://lists.berlios.de/pipermail/opensim-dev/2013-October/012029.html
> [2] http://opensimulator.org/wiki/Server_Commands#Config_Commands
>
