Karen Palen kirjoitti:
> A "mesh" cannot be uploaded yet, but you should look at the work being 
> done by the RealExtend group on this front. See 
> http://www.realxtend.org/page.php?pg=downloads for more information. 
> They presently have both an opensimulator add-on and a viewer that 
> handles meshes. However it is in very early development. The viewer 
> (Naali) and the server (Taiga) are both available for Win, Mac and Linux.

Yep, and for bringing individual meshes from sketchup there is an old 
but current tutorial at 
http://peterquirk.wordpress.com/2008/12/22/tutorial-using-the-streamlined-tool-chain-for-importing-sketchup-models-into-realxtend-04/

 From Blender, Max and Maya etc. you can also bring a full scene in one 
go, this is an old tutorial of that using the original console command 
ui 
http://realxtend.blogspot.com/2009/12/tutorial-to-import-scene-from-blender.html 
-- nowadays there is an integrated GUI for the server upload in the 
Naali viewer (we'll make a video of how that works soon).

For Blender specifically there's also an integrated UI to Blender itself 
so you can press 'publish' in Blender directly to have it upload the 
scene to an opensim+modrex server: 
https://sim.lorea.org/pg/groups/5/b2rex/ .. there's nice screenshots too 
that show the same scene in Blender & OpenSim.

I guess we should add this to some F.A.Q. on opensimulator.org so 
wouldn't have to write same emails every week :o
