I'm afraid Imprudence limits prims to 256 meters in size. An xml won't help
there, as the import would scale it back to 256 meters again. However, it is
possible to create larger megaprims. I used these steps:
1. Rez the prim
2. Give it a unique name (like "512square")
3. Search for its name in the prims table in the database; copy its UUID
4. Search for the UUID in the primshapes table. Edit the properties the way
you want.
5. Restart the region for the changes to take effect

Enjoy your gigaprim.
