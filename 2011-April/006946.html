On 01/04/11 18:41, DutchGlory wrote:
> question:
>
>
> I have 3 regions in my stand-alone.  Is it possible to make on of these
> regions
> "Currently selected" on startup? (or last visited location) now on startup
> its always "Root" i have put
> "change region" command in "startup_commands.txt" but it still going back to
> "root"

The first region config picked up becomes 'root'.  So if you only have one config file you can reorder the regions so 
that the one you want as 'root' is at the top.  If you have multiple config files it's more tricky - you'll have to 
check your log file to see the order in which they are loaded - I suspect it's alphabetical.

As far as I recall, 'root' was originally meant to mean that a command was meant to apply to all regions, with the 
individual regions being 'subdirectories'.  However, this isn't obeyed by a lot of commands and for many of them doesn't 
make any sense.
