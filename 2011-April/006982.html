On 05/04/11 10:15, Simon Slavin wrote:
> I'm running OpenSim on a university campus.  For access to web sites off-campus we /have/ to go through the campus proxy server, using an IP address and a port number.  If we try to access the web server directly we get no useful answer.
>
> In version 0.6.4 of the server there's a "HttpProxy" line in OpenSim.ini where you can put in your web proxy details.  Looking at the .ini file for current versions, I see no equivalent settings.  Does the client now use the operating system to access the web, or did the proxy settings get moved to one of the other .ini files ?  Where ?
>

Hi Simon.  The HttpProxy setting still exists in [Startup] - in the latest OpenSim you can see it commented out in 
bin/OpenSimDefaults.ini and it can be set in OpenSim.ini.

However, this is http proxy support for llHTTPRequest and dynamic textures (which you may still want).  If you want to 
change the proxy settings for the viewer, then this needs to be done in each viewer separately (under the web tab in the 
preferences dialog).  This has always been the case.
