Adam's original code was a version of the file Stitcher.cs, which 
created a bunch of sculpties and image files. It was full of hard-coded 
constants. The code that I made available made those constants 
variables, so that it's possible to handle terrains of various sizes. 
Plus, I added additional steps to import the textures directly to the 
asset server and to generate the prims that then can be loaded like this 
on the server console:
# load xml2 prims.xml

The files are still independent, so it's easy to isolate the texture 
generation from the delivery mechanism(s).
