That seems to use the chat messaging technique that the original poster asked about, things like llSay(school_comm_channel, (string)myvel);

Being able to share data for the whole swarm, pass arbitrary types as arguments (not just strings in chat messages) and get return values from function calls etc. make normal programming easily much more efficient and straightforward.

But LSL you can do without access to the server (which region modules require), and you can run the LSLs on every opensim (&sl) server without anything special. So I think mostly depends on the case, whether it is some app that can dev on own server, or something that want to distribute LSL style.
