Hi,

Thanks for posting. Some interesting things to note about the viewer.

-If you go to the site please use the 'Feedback' tab if you want things, we
have it out there so we can community/crowd source where people want to take
this
-We are using Unity3D 2.6 rather than 3.x..  3.x would give us better
performance but it also requires Grids to have a crossdomain.xml on their
website (this is a standard web file). We wanted the viewer to connect to
anyone that downloads the main opensim without needing to mess with their
grid. We may release a separate 3.X release.
-On the performance: We are slamming a lot of information to the plugin
without having the luxury of the downloaded client and all the ports. You
will therefore notice that your build quality (especially images) really
effect performance. If you want to built a region and test things think
about your image sizes the same way you would when developing a website.
What is cool with the fact it is Unity3d is that we could actually deploy
this viewer as a downloaded .exe, .dmg or widget that is only 4meg in size.
-When you use the viewer you are connecting direct to your region. This
means that you can connect to 'localhost' on a standalone grid.

<off-topic-comment>
For those looking to mess around with some cloud tech I recommend trying out
Google App Engine. we are hosting the viewer there and it provides a python
and java stack. If you have a Gmail account or use Google Apps for business
you already have access to Google Apps. It is amazing in that they
automatically take care of all the scaling for us.
</off-topic-comment>
