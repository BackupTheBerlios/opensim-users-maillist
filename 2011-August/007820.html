Kay,
I ran into a similar problem last year after I upgraded my server from Ubuntu 8.04 to 10.04.  If the machine sat idle for too long it would act as if it had fallen asleep.  The first attempt to log in (to anything - apache, sshd, etc. -  not just OpenSim) would fail because it took too long to wake up, but by the second attempt it was ready to go.
I unsuccessfully tried several combinations of system settings and bios settings to try to disable whatever energy saving 'feature' was causing the problem.  I think my particular hardware just wasn't intended to be used as an always-on headless server.
In the end I found that setting the machine to ping one of my other machines once every 3 minutes would keep it awake and allow logins to work on the first try every time.
