>I'm not sure it's actually possible to use Freeswitch for the whole 
>grid right now.  I know some code was put up for
>this but I'm not sure that it's fully implemented.


just to let you know... it is possible to use a Freeswitch voice 
server on regions attache to OSGrid.  And you can use the local 
connector method to do that and point at your own FreeSwitch 
server... as that probably the easiest to understand the setup for.

Or if you are separately running a Robust service for another grid 
you can piggy back on that service for FreeSwitch connectivity on 
your OSGrid regions by using a remote connector - useful if you use 
the same FreeSwitch service for both your own Grid and for your 
OSGrid hosted regions.
