On 6/25/2010 9:23 AM, Ai Austin wrote:
> Diva, this may seem a bit of a silly question, but I assume that we 
> still give link-region commands to Opensim.exe  or via the 
> startup-commands script ist and not in the Robust.exe console?  If its 
> to the Robust.exe console is there a startup commands equivalent to 
> let us make this automatic?

You can link-region in the sims' console and also in the robust console. 
You can also link automatically by searching the name (with that special 
syntax) on the map or by typing&clicking secondlife:// urls on the chat 
window.

Link-region's on the Robust server make the link permanent, i.e. it 
stays in the DB until you remove it. It survives reboots. (This is also 
true for standalones configured for that purpose, like the diva distro, 
when I do the 0.7 release for it)

>
> So for example will the following in Opensim.exe cause a hyperlink to 
> someone elses grid and which the Gatekeeper then will police?
>
>      link-region 6001,6001 yourgrid.com:8002:Your Region

Yes, this goes to the gatekeeper of yourgrid.com. There is a config var 
in the [Gatekeeper] section called
AllowTeleportsToAnyRegion
It's true by default, but ppl can make it false, in which case that grid 
will only allow links to the Default_Regions. So if that would be false 
in yourgrid.com, that command up there would end up ignoring the 
requested region.
