> We posted a patch to implement osGetInventoryLandmarkParams on Mantis
> http://opensimulator.org/mantis/view.php?id=5936
> It still needs some enhancement, but it's already working and useful in
lot of cases.
> What's missing?
> The function returns all values  same grid landmarks and landmarks to
hyperlinked region.
> For other external regions, we couldn't fetch region, particularly
"regionName", which is needed to output url, address, map search and region
name.
> In this case, the function returns "REGION_NOT_FOUND" instead of the
answer, to allow the script to handle it.
> Other values are returned in all cases (gatekeeper uri, position)
> Background: in our first steps, we thought we got the url successfully,
even for HG landmarks. We were wrong, it was random situation: the regions
of the first landmarks we tested were already linked on server (appearing
on console with "show hyperlinks").
> We are still looking for a way to get regionName (only value needed) for
external regions.
> We use World.GridService.GetRegionByUUID
> We tried to use code from HGEntityTransferModule, but it didn't work
(scene issues?)…
> Any idea is welcome!
