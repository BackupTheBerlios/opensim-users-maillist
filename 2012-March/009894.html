If you don't have memory issues... we run the Yellowstone regions in scisim
with 64 regions per sim and 16 sims (total of 1024 regions). We set the
view distance up so you can see a 10x10 area. That causes problems with the
current method of connecting the viewer to each of the regions (cant seem
to get connections to 100 different regions before the circuit times out).
We run the sims largely without prims, just GIS-derived terrain. It takes 2
8-core servers with 16G of memory in each. Stock opensim.

--mic
