In order to make the 'creator' names on built objects permanent, I
need to save the oar with the --home switch. Not clear on how to
implement this exactly! Quoth the wiki:

"If the --home option is specified then all names of creators from
this world will be appended with links to their home world. It is not
required that the service be operational; the information will be
added and it will be available in all worlds that import this OAR.
Example:
 save oar --home=http://mygrid.com my.oar"
http://opensimulator.org/wiki/OpenSim_Archives

So when I use that command, does the domain in the url need to
actually be the one the grid is attached to? I'm just working on my
local closed grid that isn't attached to any domain, but if I put my
own domain in there will that work or is there something else I need
to do?

I have tested this saving an oar from a 'generic' diva install
(0.7.5), then loading it onto a default soas (also 0.7.5). It seems to
work - sort of! I'm asking this because when I look at things other
people have (presumably) saved using this function, the 'creator'
field show up something like 'Avatar Name @ hg.osgrid.com' and on the
oar I've tested this with, while the correct creator name shows up
there is no '@ mygrid.com' appended. Is this working correctly and/or
has the implementation changed to drop the '@ mygrid.com' part in the
display?
