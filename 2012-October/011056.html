Hi folks,

We're working on a HUD that is taking advantage of several OSSL functions
to attach items to users without requiring permission checks, but I noticed
today every time we use one of these HUD functions, when the object
detaches it goes into the user's inventory, which isn't a good thing.

I see in LSL that there's a llAttachToAvatarTemp function that
says: Follows the same convention as llAttachToAvatar, with the exception
that the object will not create new inventory for the user, and will
disappear on detach or disconnect.

Are there any equivalents with the OSSL osForceAttach functions?

osForceAttachToAvatar
osForceAttachToAvatarFromInventory
osForceAttachToOtherAvatarFromInventory

If not, is it possible? Or is there some other way to make it so the items
don't go into the user's inventory?
