On Thu, Oct 18, 2012 at 9:10 AM, Fleep Tuque <fleep513 at gmail.com> wrote:

> Hi Joshua,
>
> It's already possible to backup your database (which includes user
> inventory) and take OAR snapshots of your regions, and these tasks can be
> scheduled and automated with the Task Scheduler in Windows and I'm assuming
> cron or something similar on unix.
>
> I used these terrific templates by WhiteStar Magic as the base for
> automating my backups on a Windows system, see:
> http://opensimulator.org/wiki/Smart_Prestart_WIN
> http://opensimulator.org/wiki/Backing_Up_WIN
>
> There are also other suggestions and more information for other OSs at
> http://opensimulator.org/wiki/Backups
>
> I have things set to run nightly and then use a separate batch script to
> copy the backup files from my server to an external mounted drive, that way
> I have copies locally on the server as well as on another disk in case of
> disk failure (which happened to FleepGrid not too long ago, see
> http://fleepgrid.com/blog/index.php/2012/03/19/fleepgrid-provisionally-back-up-following-hard-disk-failure/ for
> the gory details of that experience).
>
> Essentially, what you want to do is already possible and can be scheduled
> and automated so you don't have to do it manually - although as anyone will
> tell you, setting up backups and never checking them is a recipe for
> disaster, so be sure to set it on YOUR calendar to periodically test your
> backups and make sure you can restore everything as you expect.
>
> If you need more help setting things up, be sure too to tell us more
> details about your system (grid or standalone, what OS, etc.) so we can
> provide better assistance.
>
