see that migrating into immersive 3D will drag with it a lot of
functionalities from the client to the server side of things. This is
probably inevitable as we will need to rethink about how to build a back
end to support in an inworld UI for the Rift interface (not only for users
of our virtual worlds, but also for its creators).
