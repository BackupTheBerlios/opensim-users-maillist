Seeing this as gridserver.net:8002/app/agent/<uuid> is extremely odd, since such a path isn't used anywhere in the code. 
  There is no setting that would control this.

One might see this for a creator if an object has been loaded from certain IARs or come from the hypergrid, but seeing 
it in owner too is extremely odd.  The owner will always have a real UUID underneath so in principle having an odd name 
like this shouldn't cause other issues.

Does this happen when users create objects, users who otherwise can login with their normal names?
