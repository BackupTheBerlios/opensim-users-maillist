Well this is frustrating... the other day when I posted this question,
I was unable to save a heightmap using the console. I have no idea
why, it just wasn't outputting any file, giving no errors or
confirmations, just going to the next line with a fresh prompt. So
that's why I thought I'd try using the raw file save from the viewer.
But saving a heightmap via the console works fine today! No need now
to convert the raw to a bitmap. However, I'll go ahead and fill you in
on my pointless and time-consuming excursion...

When opening the raw file in Baliwick, the result bears no relation to
the actual terrain the file is supposed to based on. The file would
appear to be corrupt, but when uploaded back into OpenSim, it is
exactly as it should be.

When saving back to a raw file (in Baliwick), and loading that in
OpenSim, it is a mess, bearing no similarity to the original -
although looking about like what I'd expect from the image in
Baliwick.

I'm guessing the initial saved file is fine, but Baliwick is not
reading it correctly. So not much point in trying to export it to a
raster bitmap. This is what happens with any raw file I save out of
opensim and open in Baliwick. Tested saving & loading with Imprudence
and Hippo, same results.

Importing the raw file into Terragen had the same results as Baliwick
- that is, an apparently corrupt file that bore no relation to the
original world and not even any resemblance to the image Baliwick
produced.

Blender seems like a bit overkill for something that should be as
simple as converting a raw file into a raster bitmap, but since I
already had it installed I thought I'd try it out. I could find no way
to open or import a raw file.

Anyway, the problem has been solved! Sorry my computer seems to have
been playing games with me (instead of the other way around), and
thanks for the help! :)

Cheers,
~!CJ
