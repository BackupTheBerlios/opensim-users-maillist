i know there's an offline_im bundle you can download from the OSGrid 
archive but i personally couldn't get it running properly (which is very 
likely my fault).  i currently have offline IMs working in OS 0.7.2 + 
0.7.3 dev by having extracted only the messaging portion of 
OpenSim.Wiredux and creating a seperate web service.  inelegant and 
noob-hacky, but it works for those who don't want or need the other 
backend functions (nothing against the Wiredux project, it's a great 
system).

you can email me directly if you want info on the setup.  i will try to 
get back to you as quickly as possible.

side note: i'm actually working on a mini tutorial for anyone interested 
in setting up straightforward groups, profiles, search and offline 
message services using a single database/sql import and existing php 
modules (slightly customized) without having to install any extra 
backends or web interfaces (i.e. SimianGrid, Xoopensim, etc.).  i've 
pulled together what appears to be the proper components and settings 
for an average standalone or grid setup.  it's what i've been using for 
my own OS instances so i hope it works for others as well.   i'm still 
testing this to make sure it works universally (i.e. in non-Apache/AMP 
stack environments) but i should be finished in a day or two.

best to all.
