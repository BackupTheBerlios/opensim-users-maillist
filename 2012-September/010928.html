Hi folks,

I recently upgraded FleepGrid to 0.7.4 and I'm seeing this on my ROBUST
console:

11:29:33 - [PRESENCE HANDLER]: Exception in method getagents:
MySql.Data.Types.M
ySqlConversionException: Unable to convert MySQL date/time value to
System.DateT
ime

I saw one old mention of the same error on the listserv,
http://opensim-users.2152040.n2.nabble.com/attempt-to-upgrade-Diva-to-0-7-1-from-0-7-0-2-crash-on-login-td6357979.html,
but
that appeared to be related to simulator crashing, which is not happening
in my case.  However, I tried the solution suggested there, appending Allow
Zero Datetime=true to the connection string in Robust.HG.ini, and that
appears to have resolved the issue.

ConnectionString = "Data Source=localhost;Database=opensim;User
ID=opensim;Password=***;Old Guids=true;Allow Zero Datetime=true;"

Not sure if I should make a Mantis about this?  Searching for "Unable to
convert MySQL date/time value" on the Mantis brings up three old entries,
but nothing recent.  Am I the only one having this issue with 0.7.4?

Thanks,
