Has anyone considered the fact that Freeswitch is set to ONE IP URL ?  If it
had a way to roam through different IPs then it would know that the person
is going from one server to the next. Here is a test. Try making a 2 Region
SIM on one server. See if it keeps the voice going between the two. If it
does then you scored. If not then my theory went out the door here.  lol

If you then cross into another server ( computer with different IP ) and it
loose voice, then you found your problem.

UDP. It needs both to acquire a synchronization.
